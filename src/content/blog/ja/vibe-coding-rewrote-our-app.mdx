## 1. この幸せ、わけが分からない

![デスクで雑談する 2 人の同僚。エンジニア：「さっき何をやったの？」 PM：「私も分からない、でも本当に動いたんです」](/note/images/blog/vibe-coding-rewrote-our-app/vibe-coding-what-did-i-do.png)

**開発の同僚**：さっき何をやったの？

**私**：分かりません、でもなぜか動いてしまったんです。

**開発の同僚**：大丈夫、本物の幸せって元々わけが分からないものだよ。

この会話は、私と同僚の間で実際に起きたものです。

## 2. 不安こそ第一の生産力

![上司が 2 人の非エンジニアに、このアプリを書き直す担当になると伝える。デザイナーの心の声：「AI は信じている……でも自分のことはまだ信じられない」](/note/images/blog/vibe-coding-rewrote-our-app/vibe-coding-rewrite-task.png)

3 月初め、画面いっぱいに AI Coding の情報があふれていました。小紅書をスクロールしながら、他の人が Vibe Coding でさらっと作り上げた製品を眺めていると、不安が毎日のように付きまといました：**AI 時代、私には一体何ができるんだろう？**

そんなとき、上司から「絶対命令」が下りました：[Slax Note](https://slax.com/note/) を実験台にして、古いコードは全部消し、非エンジニアが AI に頼って作り直し、メンテナンスしろ、と。（補足：古いコードは実際には消していません）

このタスクが、私と UI デザイナーに降ってきました。正直に言うと、私たちは抵抗していました。上司、あなたは私を信頼している、AI も信頼している、でも私自身がまだ自分を信頼していないんです。心の中には抵抗が少し、そして興奮も少しありました。

## 3. AI は壊れていない、壊れそうなのは私

![4 コマ漫画：環境エラー、間違ったフォルダを開く、AI と一緒に白い画面を 2 時間デバッグ、その後ベテランエンジニアが数秒で bug を見つける](/note/images/blog/vibe-coding-rewrote-our-app/vibe-coding-bugs-were-me.png)

開発の同僚たちが、私たちにツールを教え始めました。彼らがターミナルや VS Code を開き、手慣れた様子でコマンドを一行ずつ打ち込むのを見ていても、私たちにはその文字列が何を意味するのか全く分かりませんでした。

一番微妙だったのは、ある種の「越境羞恥」でした：

私たちは開発者の前で AI に指示を出すのすら怖がっていました。笑われるんじゃないかと心配で。だって私たちが使う自然言語は、あまりにも素朴で、あまりにも「マグル」だったから。

プログラミングの知識が "Hello World" レベルしかない PM の私は、最初は自動運転をオンにしたみたいに、AI に乗っかって突っ走りました。数日後にはなんと、それらしいインターフェースが出来上がっていたんです。でもすぐに、現実が私を地面に何度も叩きつけてきました。

AI Coding の道で最大の障害物が、実は自分自身だったと気づきました。問題と BUG の 50% は、こんな理由でした：

- 環境設定や、ネットワークの問題を間違えていた。
- AI にしばらく書かせた結果、開いていたファイルディレクトリが間違っていた。
- App をコンパイルしたら白い画面、AI と 2 時間調査した結果、開発の大ベテランが私の PC を一瞥して言いました：「ここに、ブレークポイントを誤って置いていますね。」ソフトの使い方を知らずに踏んだ落とし穴。

## 4. 「無能な激昂」から、BUG と対話するまで

![左：激昂期、AI に向かって吼える。右：コンテキストを与え、ヒントを与え、忍耐を与える。下：チームメイトのように協力し、そしてふと気づく——「なんで AI と話すと、話し方を忘れちゃうんだ？」](/note/images/blog/vibe-coding-rewrote-our-app/vibe-coding-rocky-rage-phase.png)

年明けに 3 月下旬のベトナム旅行を予約していて、航空券をキャンセルするのも惜しく、結局、遊びながら不安を抱える羽目になりました。それでも幸い、戻ってみると、デザイナーが AI でデザイン案とほとんどのフローを完璧に再現していて、私たちは興奮しながら社内デモを行いました。

しかしその直後にやってきたのが、果てしない**BUG 修正地獄**です。

BUG は波のように押し寄せ、AI はしょっちゅう「東の壁を壊して西の壁を補修する」のような事をしていました。AI はそんなにたくさんの細部を覚えていられない。だから私は BUG 管理に着手し、優先度をつけ、リリースに影響しないものは後回しにするしかありませんでした。

AI と BUG について対話することについて、私も**「無能な激昂」**の時期を経験しました。最初は「お前は違う」「結局やれるのか、やれないんなら codex にやらせるぞ」とくらいしか言えませんでした。（実際こうやっても、トークンを余計に消費するだけで何の役にも立ちません）後で友人や同僚に教えてもらい、正しいやり方を学びました：

1. 背景を与える：AI に背景情報と再現手順を伝える。
2. ヒントを与える：ログを提供し、特定のインターフェースを調べるよう指示する。
3. 操作手順を与える：新人を指導するように、具体的な手順を教える、あるいは直接公式ドキュメントを食べさせる。

ふと気づきました：これって、以前、同僚とコミュニケーションするときの流れと全く同じじゃないか？ なのに、AI を前にした途端、私はどうして話し方を忘れてしまうんだろう？

## 5. AI に「監禁」されました

あの時期、私は本当に取り憑かれていました。ちょうど子どもも側にいなかったので、毎日仕事から帰ると Claude Code が走るのを見つめていました。寝る前ですら、まぶたが開く限り、目を細めて画面を見守り、あの "Yes/No" がポップアップするのを待っていました。

数日続けて、本当に限界が来ました。

疲れていたのは、AI のコーディングを見ること自体ではなく、いろんなウィンドウを行き来して、高頻度で判断を下し続ける消耗でした。

**AI って私を解放するためのものじゃなかったの？ どうして逆に、画面に縛り付けるの？**

## 6. 最終的に、私は「自分を許す」ことを学びました

私はいつでもどこでも Coding するのをやめ、要件を「削る」ようになりました。重要でないものは外すか後回しにする、絶対必要なものは自動化を考える。退勤したら自分を強制的に休ませる。

すると一番不思議なことが起きました：ここまで悪戦苦闘してきた今も、私はコードの根底のロジックなんて分かっていない、それなのに製品はちゃんと使えるものになっているんです。

**振り返ってみると、今回の実験が私に与えた一番大きな「改造」は 3 つあります：**

- 理解が一つ増えた：以前は、開発者が BUG を直しているのを見ても共感できませんでした。今回「東の壁を直したら西の壁が崩れる」絶望を実体験して、ようやく腑に落ちました。
- 自分に少し厳しくなった：以前は要件を出すことしかできませんでした。今は積極的に要件を削れるようになりました。重要でない機能の中には、作るのが大変で、しかも地雷を埋めやすいものがあります。
- AI との協力のコツを掴んだ：私はもう、AI を完全自動のロボットとは見ていません。ドキュメントを「直し続ける」ことで制約をかけ続けるインターンのような存在として扱っています。AI を「バカにさせない」方法を学んだことが、今回一番達成感のあった事でした。
