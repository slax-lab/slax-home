저희의 나중에 읽기 앱인 Slax Reader 의 CLI 를 출시했습니다. 흥미로운 부분은 CLI 그 자체가 아니라, 이제 여러분의 독서 라이브러리가 AI 에이전트가 호출할 수 있는 일급 도구가 되었다는 점입니다. 북마크, 목록 조회, 프롬프트로의 파이프 전부 스크립트 가능합니다.

리포지토리: [github.com/slax-lab/slax-reader-cli](https://github.com/slax-lab/slax-reader-cli) (MIT). 설치와 사용법은 아래.

---

## 왜 CLI 인가

대부분의 AI 에이전트는 무언가를 둘 곳이 없습니다.

Claude Code 에게 "이 기사를 나중을 위해 저장해 줘" 라고 요청해 보세요. 못 합니다. ChatGPT 는 URL 을 요약할 수 있지만, 내일 다시 찾을 수 있는 곳에 링크를 적어 둘 수는 없습니다. 여러분의 타임라인을 스캔하는 브라우저 에이전트는 영구 저장소를 가지지 않습니다. 모든 대화가 그것이 일어난 도구 안에 갇혀 있습니다.

CLI 는 그 간극을 메웁니다. Slax Reader 는 어떤 에이전트 — Claude Code, Claude Desktop, Codex, Gemini CLI, OpenClaw, Cursor, Windsurf — 도 읽고 쓸 수 있는 영구 독서 저장소가 됩니다.

구체적으로:

- X 타임라인을 스캔하는 브라우저 에이전트 → `reader-cli add` 가 링크를 저장.
- 글을 작성 중인 Claude Code → `reader-cli list --filter inbox` 를 호출해 오늘 저장한 항목을 자료로 끌어옴.
- 주말 스크립트가 지난 30 일간의 `reader-cli list --filter archive --json` 을 가져와 자체 파이프라인에 공급.

라이브러리가 더 이상 수동적인 북마크 더미가 아닙니다. 에이전트가 작업할 수 있는 코퍼스가 됩니다.

## 이걸로 할 수 있는 일

### 1. 요약하지 말고, 필터링하기

대부분의 "AI 가 긴 기사를 대신 읽어 줍니다" 도구는 요약을 기본값으로 합니다. 저희는 필터링이 더 유용하다고 봅니다.

Claude Code 에게 말해 보세요: *"오늘 아침에 저장한 긴 글에서 가장 직관에 반하는 3 가지와 내가 동의하지 않을 가능성이 큰 1 가지를 알려 줘."*

내부적으로 에이전트는 `reader-cli list --filter inbox --json` 을 실행해 해당 북마크를 고른 다음, `reader-cli get <id> --markdown` 으로 콘텐츠를 가져옵니다. UUID 는 보지 않아도 됩니다.

기사를 축약할 필요가 없습니다. 시간을 들일 가치가 있는지, 그리고 그 안에서 중요한 5 분이 어디인지 알아야 할 뿐입니다.

### 2. 규칙으로 일괄 저장

`reader-cli add` 는 URL 을 받지만, 흥미로운 워크플로는 에이전트에게 고르게 시키는 것입니다:

- "HN 프런트 페이지를 스캔하고, 200 점이 넘는 글은 아카이브 모드 (전체 스냅샷) 로 저장."
- "내 X 타임라인을 감시. 내가 가장 자주 저장하는 8 개 계정이 올린 링크를 저장."
- "이 Substack 출판물의 최근 30 개 글을 스캔. 제목에 'agent' 가 들어간 건 모두 저장."

이건 스크래핑입니다. 차이는, 스크래퍼를 직접 작성하지 않고 규칙을 평범한 영어로 묘사한다는 점입니다.

### 3. RSS 워크플로, 뒤집기

전통적 리더 워크플로:

1. N 개의 피드를 구독.
2. RSS 리더가 모든 것을 가져옴.
3. 헤드라인을 직접 스캔.
4. 나중에 읽기 위해 저장.
5. 실제로 읽음.

3 단계 — "헤드라인을 직접 스캔" — 가 이제 AI 의 일이 되어야 합니다.

뒤집어 보면:

1. 에이전트가 피드를 구독.
2. 에이전트가 스캔하고 요약본을 보냄.
3. 읽을 가치가 있는 3 개를 지목.
4. 에이전트가 Reader 에 저장.

직접 운영하는 구체 예:

- **오픈소스 LLM 출시 추적.** 에이전트가 HuggingFace 트렌딩, Anthropic / Mistral / Meta / Qwen / DeepSeek 의 릴리스 페이지, r/LocalLLaMA, arxiv-sanity 를 스캔. 읽을 가치가 있는 새 모델 카드나 벤치마크를 `models` 태그로 저장. 주간 요약.

이걸 설정한 뒤로 RSS 구독 대부분을 정리했습니다. 그 피드들이 나쁘기 때문이 아니라, 스캔 단계가 더 이상 내 일이 되지 않아야 하기 때문입니다.

### 4. 과거의 나에게 묻기

모든 글쓴이는 같은 바퀴를 다시 만듭니다. 지난달에 어떤 아이디어를 명확하게 설명한 글을 읽었는데, 지금 찾을 수가 없습니다.

글을 쓰기 전, Claude Code 에게 묻습니다: *"내 Slax Reader 아카이브에서 지난 90 일간 에이전트 디자인 패턴에 관한 모든 것을 찾아 줘."* 에이전트는 `reader-cli list --filter archive --json` 을 호출해 컨텍스트 안에서 필터링한 뒤, 과거의 나에게서 온 리서치 브리프를 돌려줍니다. 아카이브 전체를 직접 다시 읽지 않아도 됩니다.

**고급 활용:** 현재 가진 견해와 충돌하는 글을 읽을 때마다 `#self-challenge` 로 저장하세요. 분기에 한 번 다시 읽으세요. 그것이 쿼리 가능해진 여러분의 의견 진화입니다.

## 제품에 관한 몇 가지

회의적인 분들을 위해:

- **오픈소스, 셀프 호스팅 가능.** Slax Reader 자체는 Apache 2.0 ([github.com/slax-lab/slax-reader](https://github.com/slax-lab/slax-reader)). CLI 는 MIT. 언제든지 내보내기 가능.
- **현금 흐름이 안정적이며 장기 운영에 진심입니다.** Pocket 은 2025 년 Mozilla 가 단종시켰고, 많은 분에게 나중에 읽기 도구 상실의 트라우마를 남겼습니다. 저희는 그런 일을 겪게 하지 않겠습니다.
- **영구 백업이 무료입니다.** 북마크 + 전문 캐시, 영구 무료. Readwise, Instapaper Premium, Matter, 그리고 과거의 Pocket 은 모두 이를 유료로 합니다. 저희는 아닙니다.
- **전문 검색과 하이라이트도 무료입니다.** 같은 이유.
- **플랫폼별 파서에 지속적으로 투자합니다.** X, Threads, Substack 을 잘 지원합니다. 중국어권에서는 WeChat 공식 계정, 샤오훙슈, 웨이보, 지후 전용 파서를 갖추고 있습니다.
- **iOS, Android, 웹, 브라우저 확장.**

## 설치

```bash
# 한 단계: CLI 및 AI 에이전트 스킬 설치
npx @slax-lab/reader-cli@latest install

# 인증 (reader.slax.com → Settings → API Keys 에서 API 키 붙여넣기)
reader-cli login

# 사용
reader-cli add https://example.com --tags "models,ai" --archive
reader-cli list --filter inbox
reader-cli get <bookmark-id> --markdown
```

또는 그냥 에이전트에게 설치 가이드를 가리키고 전부 맡겨도 됩니다:

> *"Slax Reader CLI 를 설치하고 설정해 줘: https://github.com/slax-lab/slax-reader-cli/blob/main/docs/ai-agent-installation-guide.md"*
