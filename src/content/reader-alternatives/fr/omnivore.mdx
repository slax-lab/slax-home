## Pourquoi Omnivore a-t-il fermé ?

L’équipe d’Omnivore a été rachetée par ElevenLabs fin 2024 et a annoncé l’arrêt du service hébergé. Le dépôt reste en ligne, le code reste public. Mais « open source » ne suffit pas à garder une appli en vie : faire tourner un service hébergé coûte de l’argent et, sans modèle de financement viable, même les meilleures applis open source finissent au rebut.

C’est la partie de « gratuit + open source » qu’on évite souvent. Le principe est juste, mais les principes ne paient pas les factures d’infrastructure. La réponse de Slax Reader : code open source (Apache 2.0), mais une offre Pro payante pour assurer le fonctionnement dans la durée. Les membres fondateurs gardent gratuitement, pour toujours, les fonctions les plus aimées — archives permanentes, surlignages illimités, résumé IA. Le Pro finance le reste.

Nous faisons tourner notre activité depuis presque 20 ans. Nous ne partons nulle part.

Omnivore a rejoint une longue liste — [vingt ans de fermetures d’applis à-lire-plus-tard, et ce qui survit](/blog/read-it-later-history/).

## Ce que Slax Reader hérite d’Omnivore

Ce qui faisait qu’Omnivore se distinguait de Pocket / Instapaper :

### Archives permanentes, gratuites

Omnivore archivait l’article entier à la sauvegarde — vous ne stockiez pas un lien, vous stockiez le contenu. Slax Reader fait pareil. La source tombe en 404 ? Votre copie reste.

### Open source

Apache 2.0, sur [GitHub](https://github.com/slax-lab). Frontend, backend, clients mobiles : tout est public. Nous avons choisi Apache 2.0 (et non l’AGPL d’Omnivore) parce que c’est plus accueillant pour ceux qui auto-hébergent ou intègrent. Le principe est le même : vos données de lecture ne doivent pas dépendre du temps de piste d’une seule équipe.

### Des surlignages et notes qui voyagent

Omnivore proposait un export propre vers Obsidian, Notion, Logseq. Nous livrons ces ponts par étapes — ils sont sur la feuille de route. En attendant, tous vos surlignages sont visibles dans Slax et exportables à la main.

## FAQ

**Puis-je importer ma bibliothèque Omnivore ?**
L’import depuis Omnivore est sur notre feuille de route. Si vous avez un export JSON ou HTML depuis votre instance hébergée ou auto-hébergée, [laissez votre e-mail](https://r.slax.com/) — nous vous préviendrons le jour de la sortie.

**Slax Reader est-il vraiment open source ?**
Oui — Apache 2.0, sur [GitHub](https://github.com/slax-lab). Le frontend (Vue), le backend (TypeScript) et le client Android (Kotlin) sont tous publics.

**Pourquoi Apache 2.0 plutôt qu’AGPL ?**
L’AGPL est plus stricte (les modifications côté serveur doivent être ouvertes). Apache 2.0 est plus accueillante pour ceux qui auto-hébergent, forkent ou intègrent. Nous ne voulons pas compliquer la vie des gens qui veulent remixer notre travail.

**Puis-je auto-héberger Slax Reader ?**
L’auto-hébergement est sur notre feuille de route. Le code est déjà public, mais nous n’avons pas encore livré une installation Docker / en un clic propre. Si c’est un impératif pour vous, [dites-le-nous](mailto:hi@slax.com) — c’est la demande la plus fréquente des réfugiés d’Omnivore.

**Et la confidentialité ?**
Nous ne vendons pas vos données. Vos données de lecture vous appartiennent. Voir notre [politique de confidentialité](https://r.slax.com/privacy).
